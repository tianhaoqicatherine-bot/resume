# 项目背景（版本一：产品需求 / 项目介绍视角）

## 项目背景与用户痛点
大学生在寻找日常实习时，常依赖社群获取零散的岗位描述（JD）。这一投递场景存在三个主要痛点：

1. **流程繁琐且重复**：求职者需根据不同 JD 调整简历、严格按要求重命名文件（如“姓名-学校-到岗时间-最长实习时间”），并手动撰写邮件正文。
2. **人工成本高**：整个“匹配 JD - 修改简历 - 命名 - 发邮件”链路耗费大量时间。
3. **缺乏数据沉淀**：投递行为无系统记录，导致求职者在接到企业回访时，常出现信息断层，无法快速匹配对应的投递岗位。

## 自动化解决方案
为解决上述痛点，设计并实现了一套基于 AI Agent 的自动化投递与管理闭环工作流：

### 1) 智能解析与简历生成
AI 自动解析目标 JD，动态修改 LaTeX 格式的简历源码，并编译下载适配该岗位的简历版本。

### 2) 规范化处理与发送
系统按照 JD 要求的规范格式自动重命名简历文件，生成定制化邮件正文，并通过 Agent 邮箱自动完成发送。

### 3) 全链路数据追踪
投递动作完成后，系统自动提取核心信息（投递公司、岗位、时间等），将其结构化沉淀至云端表格，建立可视化的投递记录看板。
